4. FIXED ENCODING ISSUES IN show_version() METHOD
──────────────────────────────────────────────────

Fixed two types of encoding problems:

a) Checkmark Mojibake (lines 753-761):
   
   BEFORE (broken encoding):
     <li>âœ" Updated field naming...</li>
   
   AFTER (HTML entity):
     <li>&#10003; Updated field naming...</li>
   
   The checkmark symbol (✓) was incorrectly decoded causing mojibake
   characters "âœ"". Fixed by replacing with HTML entity &#10003; which
   renders correctly in all contexts.

b) Greater-Than-Or-Equal Symbol (lines 767-771):
   
   BEFORE (broken encoding):
     <li>pyserial: â‰¥3.5</li>
   
   AFTER (HTML entity):
     <li>pyserial: &gt;=3.5</li>
   
   The ≥ symbol was broken. Fixed by using HTML entities &gt;= which
   renders as >= in the interface.

Why These Changes:
------------------
- HTML entities are safe across all encodings
- No dependency on file encoding
- Renders correctly in QMessageBox
- Prevents future encoding issues
